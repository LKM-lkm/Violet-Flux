### 📖 前言

在进行网页设计或前端复刻时，我们经常会被某些网站优雅的字体所吸引。但是，一个网页可能嵌套了多层 CSS，手动在 `Elements` 面板里一个个找 `font-family` 简直是效率杀手。

今天分享一段简单的 **JavaScript 控制台代码**，只需一秒，就能帮你揪出当前网页使用的所有字体名称以及它们的原始下载链接。

---

### 🛠️ 核心原理

要彻底摸清网页的字体，我们需要从两个维度下手：

1. **Computed Style (计算样式)**：遍历 DOM 树，获取浏览器最终渲染在每个元素上的字体名称。
    
2. **@font-face 规则**：解析样式表（Stylesheets），提取远程字体文件（如 `.woff2`, `.otf`）的 URL。
    

---

### 💻 脚本代码

你可以直接将以下代码复制，粘贴到浏览器控制台（按下 `F12` -> `Console`）并回车：

JavaScript

```
(function() {
    const fonts = new Set();
    const fontFiles = new Set();

    // 1. 扫描所有元素，获取实际渲染的字体名
    document.querySelectorAll('*').forEach(el => {
        const fontFamily = window.getComputedStyle(el).fontFamily;
        if (fontFamily) {
            fontFamily.split(',').forEach(font => {
                fonts.add(font.trim().replace(/["']/g, ""));
            });
        }
    });

    // 2. 深入样式表，抓取远程字体文件链接
    for (let i = 0; i < document.styleSheets.length; i++) {
        try {
            const sheet = document.styleSheets[i];
            const rules = sheet.cssRules || sheet.rules;
            for (let j = 0; j < rules.length; j++) {
                if (rules[j] instanceof CSSFontFaceRule) {
                    const src = rules[j].style.getPropertyValue('src');
                    if (src) fontFiles.add(src);
                }
            }
        } catch (e) {
            // 提示：由于浏览器安全策略，无法读取跨域 CDN 的 CSS 内容
            console.warn("跳过受限的跨域样式表: ", e.message);
        }
    }

    // 3. 结果美化输出
    console.group("🎨 网页字体深度分析报告");
    console.log("%c📌 已使用的字体清单:", "color: #3498db; font-weight: bold;");
    console.table(Array.from(fonts).sort());
    
    console.log("%c🔗 字体文件链接 (可直接下载):", "color: #e67e22; font-weight: bold;");
    if (fontFiles.size > 0) {
        fontFiles.forEach(file => {
            const cleanUrl = file.match(/url\(['"]?([^'"]+)['"]?\)/);
            console.log(cleanUrl ? cleanUrl[1] : file);
        });
    } else {
        console.log("未检测到外部字体文件（可能是系统预装字体）。");
    }
    console.groupEnd();
})();
```

---

### 🔍 深度解析

- **`window.getComputedStyle`**: 这是获取元素“最终真相”的唯一方法，它能跳过复杂的 CSS 权重，直接告诉你浏览器最后用了什么字体。
    
- **`Set` 数据结构**: 自动去重，确保报告里不会出现 100 个重复的 "Arial"。
    
- **`CSSFontFaceRule`**: 专门用于拦截那些通过 `@font-face` 引入的 WebFont，这比去 Network 面板里人肉搜索要快得多。
    
- **安全限制 (CORS)**：如果样式表存放在不同的域名下（如 Google Fonts 的 CDN），出于隐私安全，脚本可能无法直接读取其内部规则。这时建议配合浏览器的 **Network 面板** 辅助查看。

### 💡 进阶技巧：直接下载

如果你在报告中看到了喜欢的 `.woff2` 或 `.ttf` 链接，可以直接右键点击链接选择 **"Open in new tab"**，通常浏览器会自动触发下载。
